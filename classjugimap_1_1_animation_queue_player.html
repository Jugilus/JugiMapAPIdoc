Detailed Description

The queue animation player for playing animations via animationInstance objects.

The queue player can delay a new animation in order for the old animation to complete its loop. There can be up to five animations in queue; if there are more the oldest waiting will be removed. The behavior that an animation complete its loop before enfding is set by AnimationInstance::SetCompleteLoops command. The goal of this behavior is to allow designing smooth transition beetwen animations. The AnimationQueuePlayer objects are used for playing sub-animations.
